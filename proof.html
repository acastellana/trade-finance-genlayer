Economic Model

Conditional escrow with pre-agreed remedy schedule

Importer pre-funds 73,950 PEN into conditional escrow. Release depends on shipment timing. GenLayer evaluates the disputed shipment evidence and returns the fact the contract needs in order to continue.

GenLayer decides the fact. The contract applies the consequence.

Late payment changes settlement timing. Late shipment changes whether and how much escrowed funds should be released.

The buyer must submit proof that the goods were actually returned or formally rejected in accordance with the contract before a full refund can be released. If no return proof within 14 days, defaults to 98.5% release (150 bps penalty). Return required
Evidence unclear UNDETERMINED Held in escrow — 14-day arbitrator window. If no ruling, defaults to refund. Manual review

GenLayer evaluates the shipment evidence and classifies the crossing into one of these buckets. The settlement contract maps each classification to a payout split. GenLayer never decides the price or the payment amount.

Disputed-Shipment Scenarios

Three parallel deployments — one per disputed-shipment outcome

Each scenario contract was deployed with different disputed shipment evidence to produce a different GenLayer verdict. They are not branches of one live trade.

The scenarios below reflect the currently deployed v0.1 contracts. The graduated penalty schedule above is the target economic model and will be reflected in future deployments.

Additional delay buckets (moderate delay, significant delay, very late / return-required) are defined in the contract but not yet deployed as live scenarios.

Trust Model

Two delivery paths, two trust levels

Benchmark path

GenLayer validators agree on the benchmark; delivery back to Base is relayer-mediated.

Relayer calls receiveRate() directly on Base. Protected by onlyRelayer modifier. The rate is a public market number — verifiable by anyone. No cryptographic bridge needed.

Verdict path

GenLayer evaluates the disputed shipment evidence; delivery back to Base is bridge-mediated.

Protected by onlyVerdictSource: only accepts from courtContract (Factory, called only by BridgeReceiver). Relayer cannot forge a verdict.

Verdict chain of custody

1
GenLayer contract / verdict source
Oracle evaluates disputed shipment evidence, returns classification
2
Bridge path
BridgeSender (GenLayer) → BridgeForwarder (zkSync Sepolia) → LayerZero V2
3
Base receiver / factory
BridgeReceiver (Base Sepolia) → InternetCourtFactory
4
Settlement contract action
Factory calls settlement contract with verdict → escrow releases or holds

The benchmark path is relayer-mediated (no LayerZero). The verdict path is bridge-mediated (through LayerZero). The asymmetry is intentional: wrong rate is publicly catchable; wrong classification triggers the wrong remedy. GenLayer decides the fact; the contract applies the pre-agreed economic consequence.
